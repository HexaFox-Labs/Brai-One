## Context

Индивидуальные страницы каталога были сгенерированы из коротких полей
`summary`, `whatItIs`, `purpose`, `usage`, `limitations` и `verification`. Это
обеспечивает единообразие, но повторяет компактную stack-таблицу и не даёт
читателю достаточного контекста для выбора инструмента.

## Goals / Non-Goals

**Goals:**

- сделать каждую страницу самостоятельным человеческим объяснением;
- хранить расширенное содержание в том же каноническом manifest;
- добавить фактические сценарии, границы, ошибки и связи без выдумывания
  неподтверждённых возможностей;
- сохранить детерминированную генерацию Markdown и JSON.

**Non-Goals:**

- не превращать страницы в полную upstream-документацию продукта;
- не добавлять изображения, web UI или external search в этом изменении;
- не менять категории, runtime contracts или способ установки инструментов.

## Decisions

### Narrative block внутри записи manifest

Каждая запись получает обязательный `details` block с полями `whyThisTool`,
`howItWorksHere`, `capabilities`, `scenarios`, `commonMistakes` и `relatedTools`.
Это остаётся частью одной записи, поэтому генерация не собирает описание из
нескольких источников.

`relatedTools` хранит только ids из того же manifest; generator превращает их в
ссылки и проверяет, что target существует.

### Page structure

Страница будет иметь последовательность: краткий ответ, что это, зачем в Brai,
как работает у нас, возможности, сценарии, когда применять, границы и типичные
ошибки, связанные инструменты, жизненный цикл/версия, проверка, источники и
дальнейшее чтение. Дублирование коротких полей допустимо только как навигация
между разными уровнями чтения.

### JSON parity

`docs/stack/catalog.json` переносит весь `details` block и стабильные page URLs,
чтобы будущий сайт мог отображать одинаковую глубину без Markdown parsing.

## Risks / Trade-offs

- **[Risk]** Большой manifest сложнее редактировать. → Обязательные поля
  валидируются, а generated output не редактируется вручную.
- **[Risk]** Narrative может стать рекламным и неточным. → `sources`,
  verification и ограничения остаются обязательными; текст описывает Brai New,
  а не обещания upstream-продукта.
- **[Risk]** Страницы станут слишком длинными для быстрого поиска. → В начале
  сохраняется короткий summary и metadata, а подробности разделены заголовками.
