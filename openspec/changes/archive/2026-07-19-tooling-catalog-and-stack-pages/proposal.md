## Why

Сейчас стек описан несколькими крупными таблицами и не даёт отдельной
человеческой страницы для каждого инструмента. Из-за этого установка нового
инструмента, например RTK, требует ручного обновления нескольких мест, а
будущий сайт не получает стабильный каталог и классификацию.

## What Changes

- Добавляется канонический manifest инструментов с типом, назначением,
  человеческим описанием, способом использования, версией и источниками.
- Для каждой записи автоматически генерируется отдельная Markdown-страница
  формата мини-лендинга.
- Автоматически генерируются классифицированный индекс и machine-readable
  JSON-каталог для будущего сайта.
- Добавляются команды генерации и проверки, включая проверку обязательных полей,
  уникальности, ссылок на источники и отсутствия ручного drift в generated
  output.
- Существующий RTK регистрируется первым обновлённым инструментом с его
  фактической версией, путём установки и проверкой.
- Текущий каталог `docs/stack/*.md` становится обзорной навигацией, а не вторым
  источником описаний отдельных инструментов.

## Capabilities

### New Capabilities

- `tooling-catalog`: канонический каталог инструментов, классификация,
  генерация individual pages/index/site data и fail-closed validation.

### Modified Capabilities

- Нет. Существующие нормативные контракты не меняются; добавляется новый
  documentation/tooling contract.

## Impact

- Новый manifest и generator/checker в `tools/stack/`.
- Новые generated страницы и JSON в `docs/stack/`.
- Обновление `docs/stack/README.md`, корневых commands/scripts и документационных
  правил.
- Новый permanent OpenSpec capability после синхронизации Change.
- Runtime, API и deployment-порты не меняются. Установка RTK уже отражена в
  host registry `/home/mark/DEPLOYMENT.md` и проверяется без копирования
  секретов.
