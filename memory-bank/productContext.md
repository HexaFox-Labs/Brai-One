# Product Context

## Пользователи и доверенные роли

- Platform superadmin — единственный субъект, который может изменить глобальный
  `developer_mode` пользователя.
- Project owner/admin — управляет проектом, но не выбирает runtime-профиль и не
  повышает права агента.
- Обычный пользователь — работает в одной постоянной изолированной среде,
  общей для всех его агентов и проектов.
- Developer agent — выполняет работу от имени host-пользователя `mark` в
  `/srv/projects/brai-new` и использует тот же sudo-контракт, что Codex Desktop.
- Операторы — обслуживают инфраструктуру и секреты вне checkout; их процедуры
  описываются в `docs/` и `infrastructure/`.

## Основные пользовательские сценарии

1. Пользователь открывает защищённый web-интерфейс и создаёт Activity.
2. Gateway валидирует запрос и обращается к владельцу данных через NATS.
3. Factory сохраняет Activity в своей приватной Supabase-схеме и возвращает
   подтверждённый результат.
4. Web показывает общий список Activity, обновляя первую страницу polling-ом.
5. Агент запускается только с профилем, выбранным доверенным сервером и
   связанным с текущим поколением доступа.
6. Обычный пользователь может хранить SQLite-проектную БД или запустить
   Postgres только внутри своей изолированной среды и квоты.

## Обещания продукта

- Отсутствие ложного успеха при недоступном Factory, NATS или хранилище.
- Идемпотентное создание Activity и предсказуемая cursor-пагинация.
- Разделение пользователей и платформы через OS-границы, квоты, cgroup,
  network policy и rootless container runtime.
- Отсутствие database credentials у web, Gateway, моделей и пользовательских
  sandbox-ов.
- Изменение режима доступа заменяет runtime, а не меняет права уже живого
  процесса.

## Продуктовые ограничения

- UI текущего Factory — русскоязычный, dark-only, desktop/mobile.
- Публичный трафик проходит через Caddy на 80/443; приложения не публикуют
  свои порты наружу.
- Проектные файлы checkout имеют одного штатного Unix-writer: `mark`.
- Секреты не являются частью репозиторной конфигурации.
