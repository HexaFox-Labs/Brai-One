# Memory Bank проекта

Memory Bank — это постоянный, проверяемый контекст для агентов, работающих в
`/srv/projects/brai-new`. Здесь хранится краткое состояние проекта, решения и
следующие шаги. Это репозиторная документация, а не runtime-зависимость и не
отдельный сервис.

## Порядок чтения

Используй progressive disclosure:

1. Всегда прочитай [`activeContext.md`](activeContext.md) и
   [`progress.md`](progress.md) после этого README. Это компактное ядро текущего
   handoff.
2. Для продуктовой задачи подключи [`projectbrief.md`](projectbrief.md) и
   [`productContext.md`](productContext.md).
3. Для архитектуры, границ или устойчивого паттерна подключи
   [`systemPatterns.md`](systemPatterns.md).
4. Для runtime, tooling, окружения или deployment подключи
   [`techContext.md`](techContext.md).
5. Затем прочитай только относящиеся к маршруту исходники, тесты, OpenSpec,
   reader-facing docs и операторские документы.

Если route неясен, сначала прочитай все тематические файлы, но не загружай их
автоматически для каждой задачи. Для конкретного поведения источником истины
являются код и тесты; для нормативных границ — `AGENTS.md` и постоянные
OpenSpec-спецификации.

## Как обновлять память

- В начале работы проверь `activeContext.md`, чтобы продолжить незавершённую
  задачу, а не создавать параллельную историю.
- После значимого изменения обнови `activeContext.md`: что изменено, почему,
  что проверено, что делать дальше и что блокирует работу.
- Обновляй `progress.md`, когда меняется этап, появляется результат проверки или
  фиксируется новый известный пробел.
- Не превращай эти файлы в журнал каждого коммита: историю отдельных задач
  оставляй в OpenSpec Change, task context или архиве.
- Меняй `projectbrief.md`, `productContext.md`, `systemPatterns.md` и
  `techContext.md` только при изменении устойчивого факта; рядом указывай
  ссылку на источник в репозитории.
- Все даты записывай в UTC в формате `YYYY-MM-DD`.
- Не записывай секреты, токены, пароли, приватные ключи, содержимое
  секретных env-файлов, персональные данные или большие логи.
- Не дублируй нормативные требования целиком: фиксируй краткое резюме и ссылку
  на источник.

## Приоритет источников

При расхождении документов действует следующий порядок:

1. текущие инструкции пользователя и корневой `AGENTS.md`;
2. нормативные OpenSpec-спецификации;
3. исходный код, конфигурация и тесты;
4. эта Memory Bank;
5. архивные design/proposal/task-документы.

Если Memory Bank устарела, исправь её по более приоритетному источнику и
отметь проверку в `activeContext.md` или `progress.md`.
